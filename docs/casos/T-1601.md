## Observações técnicas

- **Como a UI representa "transparência"**: o input `#water-mark-video-font-color` (Chakra, type=text, placeholder `#000000FF`, maxlength=9) aceita hex de 8 chars no formato `#RRGGBBAA`. Os 2 últimos chars são o alpha em hex (`00`=totalmente transparente, `FF`=totalmente opaco). O popover da swatch também tem slider Chakra `aria-label="Alpha"` 0-100%. Setup escreveu `#FFFFFF00` direto no input + Tab pra commitar — abordagem mais robusta que arrastar o slider.
- **Convenção 0% de transparência = totalmente transparente**: o caso usa "0% de transparência" como sinônimo de "100% transparente / 0% opaco" — confirmado pelo parêntese "(totalmente transparente)" no enunciado. Mapeia para `alpha=00` no hex.
- **Limitação da sonda DOM**: o overlay `.plyr > div[z-index:99999]` ficou com `child_count=0` neste cenário. Isso impede distinguir "feature de marca desligada" de "marca configurada mas com alpha=0 → frontend optou por não renderizar". O `setup_t1601.py` cobre esse gap verificando explicitamente que `marca enabled=True` e `alpha_chars=="00"` após salvar — então sabemos que a feature está ON. A screenshot do passo 3 também é evidência visual direta.
- **Estado pós-teste**: 9280032 fica com marca ON + cor `#FFFFFF00`. Casos seguintes que exigem marca visível (T-1595/T-1596/T-1597/T-1598) precisam rerodar seu próprio `setup_t<N>.py` antes — o de T-1595 reseta cor (na realidade não toca em cor; pode ser necessário ampliar).
