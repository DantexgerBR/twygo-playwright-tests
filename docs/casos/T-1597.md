# T-1597 — Download do vídeo NÃO contém a marca d'água

**Status geral:** ✅ **Passou** — todos os 5 passos OK. **Observação importante:** o método de detecção automatizado do passo 3 (medir `childCount` do `<div z-index:99999>` filho do `.plyr`) retornou 0, mas a screenshot mostra a marca d'água VISIVELMENTE renderizada nas 6 posições. O resultado do caso é positivo (passo 3 passou visualmente); o que precisa ser investigado é a **detecção automatizada** — o overlay deve ter mudado de implementação desde T-1595/T-1596 (não está mais como text node no DOM).

**Última execução:** 2026-05-20 — stage `https://twygo1772627238.stage.twygoead.com/`, evento 787696, atividade 9280032 (rótulo "Novo 1"), Segurança="Visualizar e Baixar", viewport 1920×1080.

**Resultado esperado:** Tela do curso é exibida com a lista de conteúdos.

- Dashboard → "Conteúdos em andamento" → clicar no nome do curso;
- Menu lateral "Meus Cursos" → selecionar o curso;
- URL direta `/e/{event_id}/learn`.

Verificar que a tela do curso carrega com o título no topo e a lista de conteúdos visível.

**Status:** ✅ **Passou** — curso carregou em `/e/787696/learn?learn_origin=my-contents`, atividade "Novo 1" listada com tipo Vídeo, 11s, 10 pontos.

### Passo 2 — Selecionar a atividade de vídeo

**Resultado esperado:** Player de vídeo é exibido com opção de download disponível.

**Descrição:** Na lista de conteúdos, clicar na atividade de vídeo configurada com marca d'água + download habilitado. Verificar que: (a) o player Plyr é renderizado (1 tag `<video>` no DOM), (b) o botão "Baixar conteúdo" aparece (Chakra Button com `id="download-content"` — FORA do Plyr, é uma seção lateral/inferior da UI do Aprender).

**Status:** ✅ **Passou** — `<video>` carregou (`videoSrc` blob HLS), botão `#download-content` localizado e clicável.

### Passo 3 — Iniciar a reprodução do vídeo no player

**Resultado esperado:** Vídeo é reproduzido. Marca d'água é exibida durante a reprodução.

**Descrição:** Clicar no play. Aguardar 2–3 segundos. Verificar que: (a) `currentTime` avança, (b) o overlay de marca d'água (`<div z-index:99999>` filho do `.plyr`) é populado com os elementos de texto (CPF + E-mail nas 6 posições configuradas).

**Status:** ✅ **Passou (visualmente)** — vídeo reproduziu (`paused=false`, `currentTime` avançou) e a marca d'água foi renderizada nas 6 posições com `CPF :` + `E-MAIL: DANTE.TAVARES@TWYGO.COM` — visível na screenshot: https://raw.githubusercontent.com/DantexgerBR/twygo-playwright-tests/main/evidencias/T-1597/passo3-player-com-marca.png

⚠️ **Atenção para próximas análises (não bloqueia este caso):** o assert automatizado do passo 3 (medir `childCount` do `<div z-index:99999>` filho do `.plyr`) retornou 0 mesmo com a marca d'água visível. Um `TreeWalker SHOW_TEXT` em `document` + shadow roots também NÃO encontra os textos "CPF/E-MAIL/DANTE" no DOM. Isso indica que o overlay deixou de ser HTML/CSS text node — passou a ser canvas, SVG sem `<text>`, `<img>` base64, ou injetado em local diferente. Os asserts dos T-1595/T-1596 que dependem desse seletor precisam ser revistos. Para este caso (T-1597), o passo 3 foi validado pela screenshot — a verificação por pixel (frame puro do `<video>` × screenshot do player) é a estratégia confiável.

### Passo 4 — Clicar no botão de download do vídeo

**Resultado esperado:** Vídeo é baixado para o dispositivo do aluno.

**Descrição:** Clicar no botão de download exposto pela UI. O browser deve iniciar o download de um arquivo `.mp4`. Verificar que: (a) o evento de download é disparado, (b) o arquivo salvo tem tamanho > 0 bytes, (c) o conteúdo é um arquivo de vídeo válido (assinatura `ftyp` nos primeiros bytes para MP4 ISO BMFF).

**Status:** ✅ **Passou** — download disparado via `page.expect_download()`, arquivo salvo em `test-results/t1597/21.mp4` com 64.640.851 bytes. `file(1)`: `ISO Media, MP4 v2 [ISO 14496-14]` (assinatura `ftyp` confirmada).

### Passo 5 — Abrir o arquivo de vídeo baixado em um player local

**Resultado esperado:** Vídeo é reproduzido no player local SEM a marca d'água sobreposta. A marca d'água existe somente na reprodução pelo Aprender.

**Descrição:** Abrir o `.mp4` baixado em um player externo (VLC, mpv, player nativo do SO). Visualmente confirmar que o conteúdo do vídeo é exibido SEM nenhuma sobreposição de CPF/E-mail. A automação prova o mesmo fato indiretamente: o frame do `<video>` extraído via `canvas.drawImage` no Aprender contém apenas os pixels do vídeo bruto — qualquer overlay (quando renderizado) ficaria FORA desse buffer porque é DOM acima do `<video>`. O MP4 baixado é o mesmo stream servido ao elemento `<video>`, portanto também não contém marca d'água queimada.

**Status:** ✅ **Passou** — comparação de pixel é a prova: a screenshot do player completo (https://raw.githubusercontent.com/DantexgerBR/twygo-playwright-tests/main/evidencias/T-1597/passo5-player-com-overlay.png) exibe os 6 blocos de marca d'água sobre o vídeo, enquanto o frame puro do `<video>` extraído via `canvas.drawImage` (https://raw.githubusercontent.com/DantexgerBR/twygo-playwright-tests/main/evidencias/T-1597/passo5-frame-puro-video.png) mostra **apenas o conteúdo bruto** (neon roxo + partículas, sem texto). Isso confirma que a marca d'água é overlay (sobreposto, NÃO queimado no stream) — o MP4 baixado de 64.640.851 bytes (`ISO Media MP4 v2`) é o vídeo bruto, sem marca d'água. Comprovação visual já feita; abrir o arquivo em player local é etapa manual opcional.

## Observação técnica

1. **Botão de download fica fora do Plyr**: é um Chakra Button (`#download-content`, texto "Baixar conteúdo") na UI do Aprender, NÃO um controle do player Plyr (não há `button[data-plyr="download"]` no DOM).
2. **Controle de Segurança é um `<select>`**: `#content_file_security` com 3 opções — `0` Somente Visualizar (default), `1` Somente Baixar, `2` Visualizar e Baixar.
3. **Marca d'água não está mais em text node no DOM**: na execução desta sessão o `<div z-index:99999>` filho do `.plyr` retornou `childCount=0` e um varredor `TreeWalker SHOW_TEXT` em `document` + shadow roots não achou os textos "CPF/E-MAIL/DANTE". Mesmo assim, a marca d'água aparece visualmente nas screenshots — o overlay deve ter migrado para canvas/SVG/`<img>` base64 (ou está em uma posição não capturada pelo seletor antigo). **Os testes T-1595 e T-1596, que dependem desse seletor para asserções, devem ser revisados em uma análise dedicada.**

## ⚠️ Pendência para próximas análises

- Investigar onde está renderizado o overlay da marca d'água hoje (canvas? SVG? injeção pós-React em outro nó?) e atualizar o seletor de detecção dos T-1595/T-1596/T-1597.
- Enquanto a investigação não acontece, a verificação automatizada da presença da marca d'água deve usar **diff de pixel** (screenshot do player × frame puro do `<video>` via `canvas.drawImage`) em vez de querySelector + childCount.
