## Observações técnicas

- **Estratégia sem-side-effects**: o teste abre o form de edição de uma atividade EXISTENTE de vídeo (9280032) e itera o radio `media_type` para cada tipo. O form re-renderiza imediatamente em React/Chakra mostrando os campos do novo tipo, mas **nada é persistido** até o Salvar — e o teste NÃO clica em Salvar. Assim, o estado da 9280032 fica intacto entre as 9 iterações.
- **Mapeamento PT → media_type (mapping do form Rails/React)**:
  - Texto → `text`
  - Página → `page`
  - Aula → `lesson`
  - PDF Estampado → `pdf` (com feature paralela "marca d'água no arquivo")
  - Vídeo (interno) → `video` (única com checkbox `#water-mark-video-enabled`)
  - Vídeo Externo → `external` (com `external_type`: youtube/vimeo/eventials/external_video)
  - Arquivos → `other` (não está no caso)
  - Questionário → `questions`
  - Scorm → `scorm`
  - Games → `games`
- **Diferença marca-vídeo vs marca-arquivo**: `media_type=pdf` mostra `Habilitar marca d'água no arquivo` (checkbox + bloco de config de cor/tamanho/posição que renderiza marca **server-side no PDF gerado**). É feature paralela e legítima — o teste valida especificamente que `#water-mark-video-enabled` está ausente, não que qualquer feature de marca esteja ausente.
- **Interpretação do caso**: as pré-condições do caso original mencionam "Aluno logado" e "atividades de arquivo previamente cadastradas com marca d'água habilitada, uma para cada formato (PDF, DOC, DOCX, XLS, XLSX, PPT, PPTX, ZIP)", mas o resultado esperado dos passos ("A opção da marca d'água NÃO vai ser exibida") só faz sentido na área admin (form de edição), não no Aprender do aluno. O teste foi implementado como **admin no form de edição** — consistente com o resultado esperado.
- **Limitação de tempo**: rodar os 9 testes em sequência (um login por teste) demora ~3 min. O fixture `admin_logado` ocasionalmente dá timeout em `wait_for_load_state networkidle` no /dashboard_students (problema do stage, não do teste). Re-rodar os falhos isoladamente sempre passa.
