Não há atividade legada real disponível no stage atual — todas as atividades de vídeo interno conhecidas (9280032 em 787696 e 9280224 em 787697) foram criadas após a feature e têm registros. **Proxy usado:** `scripts/setup_t1600.py` desmarca o checkbox de marca d'água em 9280032 ANTES do teste rodar — o comportamento de UX (checkbox desmarcado, player sem marca) é equivalente ao de uma legada do ponto de vista do usuário. A diferença é apenas no DB (registros existem com valores default vs registros ausentes), invisível pela UI.

## Observação técnica

- **Atividade-proxy vs legada real**: a 9280032 (em 787696) foi reutilizada como proxy via `setup_t1600.py` que desmarca a marca. A UX é a mesma de uma legada (checkbox off, player sem overlay), mas no DB existem registros (com `enabled=False`) — uma legada real teria ausência de registros. Para validar a diferença real no DB seria necessário SQL direto, fora do escopo da automação Playwright.
- **Ordem de execução com outros casos**: este setup deixa 9280032 com marca DESABILITADA. Casos seguintes que precisam de marca habilitada (T-1595, T-1596, T-1597, T-1598) têm seus respectivos `scripts/setup_t<N>.py` que re-habilitam. Rodar nesta ordem (ou rodar setup de outro caso depois) garante consistência.
- **Sonda do overlay**: o seletor `.plyr > div[style*="z-index:99999"]` foi mantido (mesmo padrão usado em T-1595/T-1596) — embora a armadilha conhecida (per memory) indique que esse seletor pode dar falso-negativo em alguns cenários (overlay invisível pro DOM). Aqui a sonda combinou com a screenshot visual: ambos confirmam ausência de marca. Caso futuro o seletor falhe, vencer a screenshot.
